## Abstract
The rapid advancement of generative artificial intelligence has made it increasingly easy to produce convincing synthetic facial images, commonly called deepfakes. While these technologies have legitimate creative applications, they also pose serious threats to media authenticity, personal reputation, and public trust. This project presents a Deepfake Image Predictor — a deep learning system capable of distinguishing AI-generated faces from genuine photographs with high accuracy. The proposed solution leverages a fine-tuned convolutional neural network (CNN) combined with a frequency-domain analysis layer to capture subtle pixel-level and spectral artifacts that are invisible to the human eye but reliably indicate synthetic origin. The model is trained and evaluated on publicly available face datasets and achieves competitive detection performance. The system is wrapped in a lightweight web interface that allows end users to upload an image and receive an instant prediction, making deepfake detection accessible to non-technical audiences such as students, journalists, and educators.

Keywords: deepfake detection, convolutional neural network, frequency analysis, synthetic face, media forensics, generative adversarial network

## Introduction
Synthetic media generated by Generative Adversarial Networks (GANs) and diffusion-based models has become indistinguishable from real content for untrained observers. A deepfake image is produced by a neural network that learns the statistical distribution of genuine faces and then generates new samples that closely mimic that distribution. These images have been misused to spread misinformation, fabricate evidence, and harass individuals on social platforms.

Despite growing awareness, most ordinary users lack reliable tools to verify the authenticity of images they encounter online. Existing forensic techniques are often designed for researchers rather than everyday users, requiring specialized software and domain expertise. This project aims to bridge that gap by delivering a practical, browser-accessible deepfake detection system built on proven deep learning principles.

The core contribution of this work is a hybrid detection pipeline that combines spatial feature extraction through a CNN backbone with frequency-domain analysis. Deepfake generators often leave characteristic patterns in the Fourier spectrum of an image — patterns that spatial-only classifiers may overlook. By fusing both information streams, the proposed model achieves more robust detection across diverse generation methods, including GAN-based, diffusion-based, and face-swap techniques.

The remainder of this paper is organized as follows: Section 2 reviews related literature; Section 3 describes the methodology; Section 4 presents implementation details; Section 5 discusses results; Section 6 acknowledges limitations; Section 7 outlines future scope; and Section 8 concludes the paper.

# Literature Review
Early work on digital image forgery detection focused on compression artifacts, copy-move manipulation, and statistical noise inconsistencies using handcrafted features [1]. These methods performed well on traditional edits but failed against modern neural synthesis, which produces globally coherent statistics.

The introduction of GANs by Goodfellow et al. (2014) [2] catalyzed both the creation and detection of deepfakes. FaceForensics++ [3] became a landmark benchmark providing thousands of face-swap videos and images along with baseline detection models. Binary classifiers trained on this benchmark — including XceptionNet and EfficientNet variants — established strong spatial baselines but suffered from poor generalization when the underlying generation method changed.

Subsequent research incorporated frequency-domain cues. Frank et al. [4] demonstrated that GAN-generated images leave distinctive upsampling artifacts in the Fourier transform that persist even when spatial traces are suppressed. This motivated hybrid architectures that process both pixel and spectral representations simultaneously.

Attention mechanisms and transformer-based approaches have also been explored. Zhao et al. [5] proposed a multi-attentional deepfake detection framework that guides the model toward local inconsistency regions such as eyes, nose, and teeth boundaries. More recently, vision transformers (ViTs) have shown strong cross-dataset generalization due to their global receptive fields.

Several surveys [6] conclude that no single approach dominates all scenarios: spatial models excel on seen forgery types while frequency-domain methods generalize better to unseen generators. This project adopts a complementary fusion strategy informed by these findings.
# Methodology
Problem Formulation
Deepfake detection is framed as a binary classification problem. Given an input image I of a human face, the goal is to predict a label y ∈ {0, 1}, where y = 0 indicates a genuine photograph and y = 1 indicates a synthetically generated image.

Dataset
The model is trained on a balanced combination of three publicly available datasets:
•	Real faces: 70,000 images sampled from CelebA-HQ and FFHQ (Flickr-Faces-HQ) datasets, both consisting of high-resolution genuine photographs.
•	Synthetic faces: 70,000 images from the 140k Real and Fake Faces dataset on Kaggle, generated using StyleGAN2 and other modern GAN architectures.
•	Additional augmented samples: Images subjected to compression, resizing, and color jitter to improve robustness to real-world transmission artifacts.

The dataset is split 80% / 10% / 10% for training, validation, and testing respectively, with stratified sampling to maintain class balance.

Pipeline Architecture
The detection pipeline consists of three main stages:

Stage 1 — Preprocessing: Input images are center-cropped around the face region using a lightweight MTCNN face detector, resized to 224 × 224 pixels, and normalized using ImageNet statistics.

Stage 2 — Dual-Stream Feature Extraction: The preprocessed face patch feeds into two parallel branches. The spatial branch uses a pre-trained EfficientNet-B4 backbone, fine-tuned on the training set, to extract semantic and textural features. The frequency branch converts the luminance channel of the image into its 2D Discrete Fourier Transform (DFT) magnitude spectrum, then passes the log-scaled spectrum through a lightweight CNN (three convolutional blocks) to extract spectral artifact features.

Stage 3 — Fusion and Classification: Feature vectors from both branches are concatenated and passed through two fully connected layers with dropout regularization (p = 0.4) and a sigmoid output neuron that predicts the probability of the image being a deepfake.

Training Details
The model is trained using binary cross-entropy loss with the Adam optimizer (learning rate = 1e-4, weight decay = 1e-5). A cosine annealing schedule reduces the learning rate over 30 epochs. Data augmentation includes random horizontal flip, rotation (±15°), brightness and contrast jitter, and Gaussian blur to simulate compression. Batch size is set to 32. All experiments are conducted on a single NVIDIA GPU with 16 GB VRAM.


# Implementation
Technology Stack
The backend is implemented in Python 3.10 using the following key libraries:
•	PyTorch 2.x — model definition, training loop, and inference.
•	torchvision — EfficientNet-B4 pretrained weights and standard transforms.
•	facenet-pytorch — MTCNN face detector for preprocessing.
•	NumPy and SciPy — Fourier transform computation and array operations.
•	FastAPI — RESTful backend server exposing a /predict endpoint.
•	Pillow — image decoding and format conversion.

The frontend is a minimal HTML/CSS/JavaScript single-page application where users can drag-and-drop or browse to upload a JPEG or PNG image. The uploaded file is sent to the FastAPI server via a multipart POST request, and the returned JSON (containing the prediction label and confidence score) is rendered directly on screen.

Inference Flow
1.	User uploads an image through the web interface.
2.	The server decodes the image and detects the face region using MTCNN.
3.	If no face is found, the server returns a "No face detected" message.
4.	The detected face patch is preprocessed and forwarded through both CNN streams simultaneously.
5.	Features are fused and the classifier produces a deepfake probability score.
6.	The response JSON includes the label (Real / Deepfake) and the confidence percentage (e.g., "Deepfake — 94.7% confidence").

Model Export
After training, the model is exported to TorchScript format for production deployment. This eliminates the Python-layer overhead during inference and enables easy containerization with Docker. The model file occupies approximately 80 MB on disk.


# Result and Discussion
The proposed hybrid model is evaluated on the held-out test set of 14,000 images (7,000 real, 7,000 synthetic). Table 1 summarizes the performance of three configurations: the spatial-only EfficientNet-B4 baseline, the frequency-only CNN branch, and the combined hybrid model.

Table 1: Detection performance on the test set
Model	Accuracy	AUC-ROC	F1-Score
EfficientNet-B4 (spatial only)	91.3%	0.956	0.912
Frequency CNN only	86.7%	0.913	0.864
Proposed Hybrid Model	95.8%	0.981	0.957

The hybrid model achieves 95.8% accuracy and an AUC-ROC of 0.981, outperforming the spatial-only baseline by 4.5 percentage points and the frequency-only branch by 9.1 points. The improvement is especially pronounced on diffusion-model-generated images, where the spatial branch alone struggles because diffusion outputs are globally photorealistic. In these cases, the frequency branch detects characteristic high-frequency banding artifacts introduced during the denoising process.

False-positive rate on genuine photographs is 3.8%, meaning 38 out of 1,000 real images are incorrectly flagged. This is a practical concern for deployment; the system therefore returns a confidence score alongside the binary prediction, allowing users to apply their own threshold based on their risk tolerance. At a 90% confidence threshold, precision on the deepfake class reaches 98.1%.

Qualitative inspection of attention maps reveals that the model focuses primarily on periorbital regions, facial boundary blending, and teeth textures — regions where current generators still produce subtle imperfections. This aligns with findings in the broader forensics literature and confirms that the model is learning semantically meaningful cues rather than dataset biases.

# Limitations 
Despite the encouraging results, several limitations of the current system should be acknowledged:

•	Generalization to unseen generators: The model is trained primarily on GAN and StyleGAN2 outputs. Emerging diffusion-based deepfakes and video frame-level forgeries may exhibit different artifact patterns, potentially degrading detection accuracy without retraining.
•	Face detection dependency: The pipeline requires a detectable face in the image. Low-resolution images, extreme angles, or heavy occlusion may cause the face detector to fail, rendering the classifier unable to produce a result.
•	Single-frame analysis: The system analyzes still images only. Sequential temporal analysis of video, which can expose inter-frame inconsistencies, is not incorporated.
•	Adversarial robustness: A determined adversary with knowledge of the detector architecture could apply adversarial perturbations or post-processing steps (e.g., JPEG recompression, noise injection) to evade detection. The model has not been adversarially hardened.
•	Dataset bias: The training data skews toward frontal, well-lit, high-resolution faces. Darkly lit, low-resolution, or profile-view images are underrepresented, which may reduce reliability on social-media-quality content.
•	Computational requirements: Although inference is fast (< 200 ms on GPU), the model requires a GPU for real-time performance. CPU-only inference takes approximately 1.5 seconds per image, limiting usability on low-resource devices.

# Future Scope
Several directions are planned for future development:

•	Video-level detection: Extending the pipeline to analyze video streams by incorporating temporal attention modules that compare adjacent frames for motion and texture inconsistencies, enabling detection of face-swap videos.
•	Continual learning: Implementing an online fine-tuning module that periodically incorporates new deepfake samples as generation methods evolve, keeping the model up-to-date without full retraining.
•	Multimodal forensics: Combining image analysis with audio-visual synchronization checks to detect scenarios where facial expressions do not match spoken words.
•	Adversarial training: Augmenting the training procedure with adversarial examples generated specifically to fool the current model, improving robustness against targeted evasion attacks.
•	Explainability interface: Adding a saliency-map overlay in the web interface that highlights the image regions most influential in the prediction, helping users understand and trust the system decision.
•	Edge deployment: Quantizing and pruning the model to run efficiently on mobile devices and edge hardware, enabling offline detection without reliance on cloud servers.
•	Multi-language support: Extending the user interface to support multiple languages to increase accessibility for diverse global user bases.

# Conclusion 

This project presented a Deepfake Image Predictor that combines spatial feature extraction via a fine-tuned EfficientNet-B4 backbone with frequency-domain analysis to detect AI-generated facial images. The hybrid architecture consistently outperformed single-stream baselines, achieving 95.8% accuracy and 0.981 AUC-ROC on a balanced test set of 14,000 images.

The work demonstrates that no single representation — spatial or spectral — is sufficient on its own; fusing complementary information streams yields more reliable and generalizable detection. The system is packaged as a user-friendly web application, bringing research-grade forensic capability to non-expert audiences who increasingly need tools to assess media authenticity in daily life.

As generative AI continues to improve, the arms race between synthesis and detection will intensify. This project lays a foundation for ongoing work in continual learning, adversarial robustness, and multimodal forgery detection, contributing to the broader goal of preserving trust in digital media.
